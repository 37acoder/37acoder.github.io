# 标准库-context-goroutine间同步利器

context库可谓是golang的除错误处理以外的一个显著特征。在golang实现的网络框架、中间件中，非常常见这个context，context常常作为各种函数的第一个参数传递进来。context库内所有context都实现了下面的接口。context以一种递归的关系来组织起来。

type Context interface {
	Deadline() (deadline time.Time, ok bool)
	Done() <-chan struct{}
	Err() error
	Value(key interface{}) interface{}
}
  • Deadline方法返回当前context的到期时间，当未设置到期时间时，ok为false。
  • Done方法返回一个channel，用来通知各context的持有者，该context已经结束
  • Err方法返回一个错误，存储context被取消的错误的原因
  • Value输入一个key并返回一个空接口，可以用来传递一些值

# 根context

根context往往是从某个调用链条的入口开始初始化，最初的context可以被称为context。根context可以通过context.Backgroud()来创建，该函数返回了一个emptyctx类型，该类型实现了Context接口，但实现的方法都是空实现。

# 子context、功能context

官方的Context接口的各种实现通过组合方式的构建。

  1. ValueContext：它组合(内嵌)了一个Context接口，重写了Value()方法。Value()方法判断当前key是否等于输入的key，如果不想等则递归的调用组合的Context的Value方法。
type valueCtx struct {
	Context
	key, val interface{}
}

func (c *valueCtx) Value(key interface{}) interface{} {
	if c.key == key {
		return c.val
	}
	return c.Context.Value(key)
}
